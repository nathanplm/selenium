Selenium IDE

Selenium IDE a été conçu pour permettre aux développeurs et aux testeurs d’enregistrer leurs interractions avec le navigateur permettant par la suite de rejouer un scénario d’interactions pour simuler un processus fonctionnel à tester.


  • C'est une extension pour les navigateurs (comme Chrome ou Firefox) qui permet notamment de créer rapidement des tests fiables de bout en bout. Elle fonctionne immédiatement pour n'importe quelle application Web.

  • Selenium IDE propose un débogage de test plus facile avec des fonctionnalités IDE (Integrated Development Environment ou Environnement de Développement Intégré en français) telles que l'utilisation de points d'arrêt, de pauses durant les tests, ou encore d'exceptions.

  • Vous pouvez également exécuter vos tests sur n'importe quelle combinaison navigateur / OS en parallèle avec la commande "Command-line Runner" de Selenium IDE.

Selenium WebDriver

Selenium WebDriver est une API compacte orientée object qui est conçue comme une interdace de programmation simplifiée.

Cela permet notamment de "piloter" un navigateur de manière native comme le ferait un utilisateur classique, soit localement, soit sur une machine distante utilisant le serveur Selenium.

Selenium WebDriver fait référence à la fois aux différents langages de programmation mais également aux implémentations du code de contrôle du navigateur utilisé.

Selenium Grid

Selenium Grid permet d'exécuter des scripts WebDriver sur des machines distantes en routant les commandes envoyées par le client vers des instances de navigateur distantes. En fait, cela permet d'exécuter de manière simple des tests en parallèle sur plusieurs machine.

Selenium Grid permet également de gérer différentes versions et configurations d'un navigateur de manière centralisée, ce qui évite de devoir gérer chaque test de manière individuelle.

Cependant, Selenium Grid possède des limites ce qui pourrait l'amener à ne pas pouvoir répondre à certains besoins spécifiques comme par exemple la gestion de l'infrastructure.

Néanmoins, voici quelques unes de ses principales fonctionnalités :

  • Point d'entrée central pour tous les tests
  • Gestion et contrôle des nœuds / environnement où les navigateurs s'exécutent
  • Mise à l'échelle
  • Exécution de tests en parallèle
  • Tests multiplateformes
  • Equilibrage de charge
