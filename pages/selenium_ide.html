Selenium IDE

Description :

Selenium IDE a été conçu pour permettre aux développeurs et aux testeurs d’enregistrer leurs interractions avec le navigateur permettant par la suite de rejouer un scénario d’interactions pour simuler un processus fonctionnel à tester.


  • C'est une extension pour les navigateurs (comme Chrome ou Firefox) qui permet notamment de créer rapidement des tests fiables de bout en bout. Elle fonctionne immédiatement pour n'importe quelle application Web.

  • Selenium IDE propose un débogage de test plus facile avec des fonctionnalités IDE (Integrated Development Environment ou Environnement de Développement Intégré en français) telles que l'utilisation de points d'arrêt, de pauses durant les tests, ou encore d'exceptions.

  • Vous pouvez également exécuter vos tests sur n'importe quelle combinaison navigateur / OS en parallèle avec la commande "Command-line Runner" de Selenium IDE.


Comment l'utiliser ?

Etant une extension d'un navigateur, Selenium IDE ne nécessite aucune configuration particulière à part l'installation de celle-ci.

Il est donc très facile et très rapide d'utilier Selenium IDE sur vos applications et de mettre en place des tests.



Pré-requis :

Pour utiliser Selenium IDE, il suffit de télécharger l'extension dans le magasin d'extensions de votre navigateur favori.



Son fonctionnement :

Tests résilients

Lorsqu'il est lancé, Selenium IDE permet d'enregistrer plusieurs "localisateurs" pour chaque élèment de la page avec lequel il interagit.

Comme visible sur l'image ci-dessus, chaque élèment de la page est "localisable" de par son attribut "target" qu'il est possible de définir vous même ou en utilisant l'inspecteur du navigateur.

Enregistrement de l'écran

En cliquant sur le bouton "REC", il est possible d'enregistrer son écran, c'est-à-dire que Selenium IDE se mettra à créer un nouveau script de test et le mettra à jour à chaque action que vous entreprendrez sur la page, le site, ou l'application sur laquelle vous vous trouvez.

Après avoir défini l'URL de départ, vous serez dirigé vers cette URL et vous pourrez voir un message en bas de votre écran vous indiquant que Selenium IDE est en train d'enregistrer ce que vous faites.

Une fois l'enregistrement coupé, vous aurez à votre disposition toutes les étapes retraçant tout ce que vous aurez fait sur la page ou l'application, et la possibilité d'exploiter ces données.

Réutilisation des tests

Grâce à la commande "Run" il est possible de lancer l'exécution d'un script de tests où même de réutiliser un cas de test précis à l'intérieur d'un autre (par exemple en utilisant uen logique de connexion à plusieurs endroits dans une suite logique).

Il est possible d'exécuter le script de tests à tout moment en appuyant sur les boutons "Run" comme montré ci-dessus ou en utilisant directement la commande "run".

Flux de contrôle

Il est également possible d'utiliser le "Control Flow" ou Flux de contrôle fourni par Selenium IDE pour réaliser des schémas ou suites de tests en utilisant notamment la multitude de commandes disponibles comme if, while ou times.

Il est donc possible d'ajouter de la logique conditionnelle ou des boucles sur les différents tests afin par exemple d'exécuter certaines commandes uniquement quand certaines conditions de votre application sont remplies ou au de réitérer certaines commandes à plusieurs reprises en fonction de critères définis.

Exportation de code

Enfin, il vous est possible d'exporter un test ou une suite de tests. Pour cela, il vous suffit de faire un clic droit sur un test et de choisir dans quel langage vous voulez l'exporter.

Cela peut vous permettre d'exploiter encore plus ces scripts de tests et de pouvoir si besoin les intégrer à un projet externe.
